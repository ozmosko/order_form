# Nitank Order Form - מדריך תפעול

## מה יש כאן

| תיקייה | שימוש |
|--------|-------|
| `deploy/` | הטופס הציבורי - מועלה ל-Netlify |
| `admin/` | ממשק הניהול - רץ מקומית בדפדפן |

---

## ארכיטקטורה

```
לקוח ← קישור אישי ← עובד ← ממשק ניהול (admin.html)
                                      ↓
                               config.js (עובדים + מוצרים)
                                      ↓
                         GitHub (main) → merge → (production) → Netlify
```

- **הטופס** נגיש רק דרך קישור אישי שכולל מזהה לקוח. פתיחה ישירה מציגה שגיאה.
- **הסיסמה** של כל לקוח היא מספר הלקוח שלו.
- **עובדים** מתחברים לממשק הניהול עם מספר הטלפון שלהם ויכולים לייצר קישורים בלבד.
- **מנהל מאסטר** מתחבר עם סיסמת המנהל ויש לו גישה מלאה.

---

## פריסה ראשונה ל-Netlify

### שלב 1 - הכן את הסביבה
1. צור חשבון GitHub וצור repository
2. העלה את תיקיית הפרוייקט כולה ל-GitHub
3. צור שני ענפים: `main` (עבודה שוטפת) ו-`production` (מה שרץ בפועל)
4. צור חשבון [Netlify](https://netlify.com)

### שלב 2 - חבר Netlify ל-GitHub
- לחץ **Add new site → Import an existing project**
- בחר GitHub ואפשר גישה ל-repository
- הגדר:
  - **Branch to deploy:** `production`
  - **Publish directory:** `deploy`
- לחץ **Deploy site**

### שלב 3 - הגדר משתני סביבה ב-Netlify
לך ל: **Site configuration → Environment variables** והוסף:

| Key | Value |
|-----|-------|
| `MAKE_WEBHOOK_URL` | כתובת ה-Webhook של Make.com לשמירת הזמנות |

### שלב 4 - פרוס מחדש
לאחר הוספת משתנה הסביבה לחץ **Trigger deploy** ב-Netlify.

## הגדרות ראשוניות של ממשק הניהול

פתח את `admin/admin.html` בדפדפן (לחיצה כפולה על הקובץ).

בהתחברות ראשונה עם סיסמת המנהל, פתח את **הגדרות** והזן:
- **כתובת הטופס הציבורי** - הכתובת שקיבלת מ-Netlify (לדוגמה: `https://your-site.netlify.app`)
- **Webhook לטעינת לקוחות** - כתובת ה-Webhook של Make.com שמחזיר את רשימת הלקוחות
- **Webhook למעקב עובדים** - כתובת ה-Webhook של Make.com לרישום פעילות עובדים (אופציונלי)

לאחר שמירה, לחץ **הורד config.js** בטאב המוצרים כדי לשמר את ההגדרות בקובץ.

## ניהול שוטף

### הוספה/הסרה של עובדים
1. פתח `admin/admin.html` והתחבר עם סיסמת המנהל
2. עבור לטאב **עובדים**
3. הוסף/הסר שורות ולחץ **הורד config.js**
4. החלף את `admin/config.js` **ו**-`deploy/config.js` בקובץ שהורדת
5. [פרוס ל-Netlify](#תהליך-פריסה)

### עדכון רשימת מוצרים
1. פתח `admin/admin.html` והתחבר עם סיסמת המנהל
2. עבור לטאב **מוצרים**
3. ערוך את הרשימה ולחץ **הורד config.js**
4. החלף את `admin/config.js` **ו**-`deploy/config.js` בקובץ שהורדת
5. [פרוס ל-Netlify](#תהליך-פריסה)

> **שים לב:** אל תשנה מזהי מוצרים קיימים - שינוי מזהה ישבש הזמנות ישנות ב-Make.com

### יצירת קישור ללקוח
1. פתח `admin/admin.html` (מנהל מאסטר **או** עובד)
2. עבור לטאב **לקוחות**
3. לחץ על כפתור ההעתקה ליד שם הלקוח
4. שלח את הקישור ללקוח - הסיסמה שלו היא **מספר הלקוח** שלו

## תהליך פריסה

פריסה ל-Netlify מתבצעת **רק** דרך הענף `production`:

```bash
# לאחר שינויים ב-main
git checkout production
git merge main
git push origin production
git checkout main
```

Netlify מזהה push ל-`production` ומפרוס אוטומטית.

> עובדים **חדשים** יוכלו להתחבר רק לאחר פריסה של `config.js` המעודכן ל-Netlify.

## הפצת ממשק הניהול לעובדים

העלה לתיקיית SharePoint משותפת את הקבצים:
- `admin/admin.html`
- `admin/config.js`
- `admin/order-form-console.bat` (קובץ הפעלה - לחיצה כפולה פותחת את הממשק)

לאחר כל עדכון של `config.js` - החלף את הקובץ בתיקיית SharePoint.
